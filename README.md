# SoftRender — CPUソフトウェアレンダラー

## 基本情報

| 項目 | 内容 |
|------|------|
| 制作形態 | 個人制作 |
| 使用言語 | C++（標準ライブラリのみ） |
| 対象環境 | Windows（Win32 API でウィンドウ描画） |
| ジャンル | 3Dグラフィックス／レンダリング技術デモ |
| プレイ動画 | https://www.youtube.com/watch?v=v55xjXCK1TE |

## 作品概要

OpenGL や DirectX といったグラフィックスAPI、および GPU を一切使わず、3Dレンダリングのパイプライン全体を CPU 上にゼロから実装したソフトウェアレンダラーです。3Dモデルを読み込み、頂点変換からクリッピング、ラスタライズ、シェーディング、深度テストを経て、最終的なピクセルを Win32 のウィンドウへ描画するまでの一連の流れを、自作のコードのみで完結させています。

設計の方針として、**OpenGL の API 体系をあえて踏襲**しました。バッファオブジェクト・頂点属性・シェーダープログラム・カリング・深度関数といった概念を OpenGL に倣ったインターフェースで再現しており、「GPU が内部で何を行っているか」をコードとして自分の手で再構築することを目的としています。

## アピールポイント（技術的に力を入れた点）

- **OpenGL 風 API を CPU 上に自作** — `bindBuffer` / `bufferData` / `vertexAttributePointer` / `useProgram` / `cullFace` / `depthFunc` / `drawElement` といった呼び出しを、`ARRAY_BUFFER`・`FRONT_FACE_CCW`・`DEPTH_LESS` など OpenGL に倣った定数とともに再現し、グラフィックスAPIの内部動作をコードで理解できる構造にしています。

- **数学ライブラリを完全に自作** — 4×4行列による平行移動・回転・スケール、透視投影行列、ビュー行列（lookAt）を、標準ヘッダ（`<iostream>` / `<assert.h>`）のみを用いてゼロから実装。GLM 等の外部数学ライブラリは使用していません。

- **プログラム可能なシェーダーステージ** — 頂点シェーダーとフラグメントシェーダーを基底クラスとして設計し、用途別に `defaultShader`（基本）・`textureShader`（テクスチャ）・`lightShader`（ライティング）の3種を実装。GPU のシェーダーモデルを模した拡張可能な構成にしています。

- **Sutherland-Hodgman クリッピング** — near / far / left / right / top / bottom の視錐台6平面に対して同次クリップ空間でポリゴンを分割し、画面外にはみ出る三角形を正しく再構成（三角形の縫合）しています。

- **透視補正付き属性補間** — スクリーン空間での単純な線形補間ではなく、透視除算と透視回復（`perspectiveDivision` / `perspectiveRecover`）を通して、奥行きを考慮した正しい属性補間を行っています。

- **ライティング** — 平行光源による拡散反射（ディフューズ）を実装。法線ベクトルと光線方向の内積で陰影を計算し、テクスチャ色と乗算して最終色を決定しています。

- **背面カリングと深度テスト** — 面の向き（時計回り／反時計回り）に基づいて裏面を破棄し、深度バッファによる前後判定（`DEPTH_LESS` 等）で隠面消去を行っています。

- **テクスチャサンプリング** — 最近傍／バイリニアのフィルタリングと、リピート／ミラーのラップモードを実装しています。

- **フレームバッファ → Win32 描画** — レンダリング結果は自作のフレームバッファ（カラー＋深度）に書き込み、Win32 の `DIBSection` を介してウィンドウへ転送しています。

## 担当箇所

個人制作のため、設計から実装まで全工程を担当しています。数学ライブラリおよびレンダリングパイプライン（頂点処理・クリッピング・ラスタライズ・シェーディング・深度テスト・フレームバッファ）は**すべて自作**です。

なお、モデルファイルの読み込みには **assimp**、画像（テクスチャ）のデコードには **stb_image** という外部ライブラリを利用していますが、これらはあくまで「アセットの読み込み」専用であり、レンダリング処理そのものには一切関与していません。

## フォルダ構成

```
SoftRender/
├── main.cpp              … エントリポイント（モデル読み込み・行列設定・描画ループ）
├── CMakeLists.txt
│
├── math/                 … 自作の数学ライブラリ
│   ├── vector.h          …   ベクトル（Vec2 / Vec3 / Vec4）
│   ├── matrix.h          …   4×4 行列
│   └── mathFunctions.h   …   translate / rotate / scale / lookAt / perspective 等
│
├── gpu/                  … レンダリングパイプラインの中核
│   ├── gpu.cpp/.h        …   パイプライン統括（OpenGL 風 API 本体）
│   ├── vao.cpp/.h        …   頂点配列オブジェクト
│   ├── bufferObject.cpp/.h …  頂点／インデックスバッファ
│   ├── clipper.cpp/.h    …   Sutherland-Hodgman クリッピング
│   ├── raster.cpp/.h     …   三角形・直線のラスタライズと属性補間
│   ├── shader.cpp/.h     …   シェーダー基底クラス
│   ├── shader/           …   defaultShader / textureShader / lightShader
│   ├── texture.cpp/.h    …   テクスチャサンプリング（フィルタ／ラップ）
│   ├── frameBuffer.cpp/.h …  カラー／深度バッファ
│   └── dataStructures.h  …   頂点出力・属性記述子等の共通構造体
│
├── application/          … ウィンドウ管理・入力
│   ├── application.cpp/.h …  Win32 ウィンドウ生成・メッセージループ・DIB 描画
│   ├── camera.cpp/.h     …   カメラ（ビュー行列・投影行列）
│   ├── model.cpp/.h      …   モデル読み込み（assimp 経由）
│   ├── image.cpp/.h      …   画像読み込み（stb_image 経由）
│   └── stb_image.h       …   画像デコード用の外部ヘッダ（同梱）
│
└── global/
    └── base.h            … 共通定義（OpenGL 風定数・数学マクロ）
```

## ソースコードの読みどころ

短時間でコードの要点を把握いただけるよう、特に見ていただきたいファイルを案内します。

| ファイル | 読みどころ |
|----------|------------|
| `gpu/gpu.cpp` | レンダリングパイプラインの中核。OpenGL 風 API の実装、頂点シェーダステージの駆動、透視除算・スクリーンマッピングまでの頂点処理の流れ |
| `gpu/clipper.cpp` | 視錐台6平面に対する Sutherland-Hodgman クリッピングと、クリップ後の三角形再構成 |
| `gpu/raster.cpp` | 三角形のラスタライズと、透視補正を考慮した属性補間 |
| `gpu/shader/lightShader.cpp` | 平行光源による拡散反射の計算（法線と光線方向の内積） |
| `math/mathFunctions.h` | 透視投影行列・ビュー行列（lookAt）・回転／平行移動／スケールなど、行列演算の自作実装 |
| `main.cpp` | モデル読み込みからモデル行列の設定、描画ループまでの全体像 |

## 動作環境・ビルド方法

- **プラットフォーム：** Windows（Win32 API を使用）
- **ビルド：** CMake ＋ Visual Studio 2022（x64-Release を推奨）
- **外部ライブラリの依存：**
  - **assimp** — モデルファイルの読み込み（vcpkg 等で導入可能）
  - **stb_image** — 画像デコード（ヘッダのみ、`application/` に同梱）

> **注記：** 本リポジトリには提出サイズの都合上、assimp 本体および 3Dモデル・テクスチャ等のアセットは含めていません。実際の描画結果は提出した[プレイ動画](〔URL〕)にてご確認いただけます。

## 補足

本作はグラフィックスAPIに頼らず一から実装することで、3Dレンダリングの仕組みを根本から理解することを目的とした技術デモです。レンダリング処理（数学ライブラリ・パイプライン）はすべて自作しており、外部ライブラリの利用はモデル読み込み（assimp）と画像デコード（stb_image）に限定しています。
