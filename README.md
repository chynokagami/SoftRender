■ 作品概要
　OpenGL や DirectX といったグラフィックスAPI、および GPU を一切使わず、
　3Dレンダリングのパイプライン全体を CPU 上にゼロから実装したソフト
　ウェアレンダラーです。3Dモデルを読み込み、頂点変換からクリッピング、
　ラスタライズ、シェーディング、深度テストを経て、最終的なピクセルを
　Win32 のウィンドウへ描画するまでの一連の流れを、自作のコードのみで
　完結させています。
　設計の方針として、OpenGL の API 体系をあえて踏襲しました。バッファ
　オブジェクト・頂点属性・シェーダープログラム・カリング・深度関数と
　いった概念を OpenGL に倣ったインターフェースで再現しており、「GPU が
　内部で何を行っているか」をコードとして自分の手で再構築することを
　目的としています。

■ アピールポイント（技術的に力を入れた点）
　・OpenGL 風のAPIをCPU上に自作。bindBuffer / bufferData /
　　vertexAttributePointer / useProgram / cullFace / depthFunc /
　　drawElement といった呼び出しを、ARRAY_BUFFER・FRONT_FACE_CCW・
　　DEPTH_LESS など OpenGL に倣った定数とともに再現し、グラフィックス
　　APIの内部動作をコードで理解できる構造にしています。
　・行列・ベクトルの数学ライブラリを外部に依存せず完全に自作。
　　4×4行列による平行移動・回転・スケール、透視投影行列、ビュー行列
　　（lookAt）を、標準ヘッダ（<iostream> / <assert.h>）のみを用いて
　　ゼロから実装しています。GLM 等の数学ライブラリは使用していません。
　・プログラム可能なシェーダーステージを自作。頂点シェーダーと
　　フラグメントシェーダーを基底クラスとして設計し、用途別に
　　defaultShader（基本）・textureShader（テクスチャ）・lightShader
　　（ライティング）の3種を実装。GPUのシェーダーモデルを模した拡張可能
　　な構成にしています。
　・クリッピングを Sutherland-Hodgman アルゴリズムで実装。near/far/
　　left/right/top/bottom の視錐台6平面に対して同次クリップ空間で
　　ポリゴンを分割し、画面外にはみ出る三角形を正しく再構成（三角形
　　の縫合）しています。
　・透視投影に対応した正しい補間を実装。スクリーン空間での線形補間
　　ではなく、透視除算と透視回復（perspectiveDivision /
　　perspectiveRecover）を通して、奥行きを考慮した属性補間を行って
　　います。
　・ライティングは平行光源による拡散反射（ディフューズ）を実装。
　　法線ベクトルと光線方向の内積で陰影を計算し、テクスチャ色と乗算
　　して最終色を決定しています。
　・背面カリングと深度テストを実装。面の向き（時計回り／反時計回り）
　　に基づいて裏面を破棄し、深度バッファによる前後判定（DEPTH_LESS
　　等）で隠面消去を行っています。
　・テクスチャサンプリングを自作。最近傍／バイリニアのフィルタリング
　　と、リピート／ミラーのラップモードを実装しています。
　・レンダリング結果は自作のフレームバッファ（カラー＋深度）に書き込み、
　　Win32 の DIBSection を介してウィンドウへ転送しています。

■ 担当箇所
　個人制作のため、設計から実装まで全工程を担当しています。
　数学ライブラリおよびレンダリングパイプライン（頂点処理・クリッピング・
　ラスタライズ・シェーディング・深度テスト・フレームバッファ）は、
　すべて自作です。
　なお、モデルファイルの読み込みには assimp、画像（テクスチャ）の
　デコードには stb_image という外部ライブラリを利用していますが、
　これらはあくまで「アセットの読み込み」専用であり、レンダリング処理
　そのものには一切関与していません。

■ フォルダ構成（主なもの）
　main.cpp        … エントリポイント。モデル読み込み・行列設定・
　                   描画ループの統括。
　math/           … 自作の数学ライブラリ（行列・ベクトル・各種関数）。
　gpu/            … レンダリングパイプラインの中核（後述）。
　application/    … ウィンドウ管理・カメラ・モデル読み込み・画像処理。
　global/         … 共通定義（OpenGL風の定数・数学マクロ 等）。
　CMakeLists.txt  … ビルド設定。

　＜gpu/ 内の主なモジュール＞
　　gpu.cpp        … パイプライン全体の統括。OpenGL風APIの本体、
　　                  頂点シェーダ起動・透視除算・画面マッピングを担当。
　　vao.cpp        … 頂点配列オブジェクト（属性レイアウトの管理）。
　　bufferObject.cpp … 頂点／インデックスバッファの管理。
　　clipper.cpp    … Sutherland-Hodgman による視錐台クリッピング。
　　raster.cpp     … 三角形・直線のラスタライズと属性補間。
　　shader.cpp     … シェーダー基底クラス。
　　shader/        … defaultShader・textureShader・lightShader。
　　texture.cpp    … テクスチャのサンプリング（フィルタ／ラップ）。
　　frameBuffer.cpp … カラー／深度バッファの管理。

■ ソースコードの読みどころ ※プログラマ向け
　短時間でコードの要点を把握いただけるよう、特に見ていただきたい
　ファイルを以下に案内します。
　・gpu/gpu.cpp
　　　… レンダリングパイプラインの中核。OpenGL風APIの実装、頂点
　　　　シェーダステージの駆動、透視除算・スクリーンマッピングまでの
　　　　頂点処理の流れ。
　・gpu/clipper.cpp
　　　… 視錐台6平面に対する Sutherland-Hodgman クリッピングと、
　　　　クリップ後の三角形再構成。
　・gpu/raster.cpp
　　　… 三角形のラスタライズと、透視補正を考慮した属性補間。
　・gpu/shader/lightShader.cpp
　　　… 平行光源による拡散反射の計算（法線と光線方向の内積）。
　・math/mathFunctions.h
　　　… 透視投影行列・ビュー行列（lookAt）・回転／平行移動／スケール
　　　　など、行列演算の自作実装。
　・main.cpp
　　　… モデル読み込みからモデル行列の設定、描画ループまでの全体像。

■ 動作環境・ビルド方法
　・対象プラットフォーム：Windows（Win32 API を使用）
　・ビルド：CMake ＋ Visual Studio 2022（x64-Release を推奨）
　・外部ライブラリの依存：
　　　- assimp（モデルファイルの読み込み。vcpkg 等で導入可能）
　　　- stb_image（画像デコード。ヘッダのみ、application/ に同梱）
　・本リポジトリには提出サイズの都合上、assimp 本体および 3Dモデル・
　　テクスチャ等のアセットは含めていません。実際の描画結果は提出した
　　プレイ動画にてご確認いただけます。

■ 補足
　本作はグラフィックスAPIに頼らず一から実装することで、3Dレンダリング
　の仕組みを根本から理解することを目的とした技術デモです。レンダリング
　処理（数学ライブラリ・パイプライン）はすべて自作しており、外部ライブ
　ラリの利用はモデル読み込み（assimp）と画像デコード（stb_image）に
　限定しています。
